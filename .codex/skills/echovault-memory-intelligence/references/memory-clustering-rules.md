# EchoVault Memory-Clustering Rules

Memory clustering groups echoes that belong together emotionally, thematically, or temporally.

## Cluster signals

Use a blend of:

- Emotional family similarity.
- Shared tags, people, places, or motifs.
- Time proximity.
- Repeated phrases or semantic concepts.
- Similar intensity and energy.

## Cluster types

| Type | Meaning | Example label |
| --- | --- | --- |
| motif | repeated symbol, place, person, or object | "Walks home" |
| emotional loop | same emotion returning across multiple echoes | "Sunday heaviness" |
| transition | before/after period around a change | "New job threshold" |
| repair | difficult emotion followed by care/growth | "After the argument" |
| quiet cluster | low-frequency but repeated silence or numbness | "Muted mornings" |

## Rules

- Do not merge unrelated intense echoes just because they are intense.
- Keep clusters small enough to feel personal and inspectable.
- Name clusters from user language when possible.
- Give every cluster a confidence score.
- Allow users to split, rename, hide, or delete clusters.
- Treat silence as a weak signal unless paired with surrounding echo context.

## Reflection use

Good cluster sentence:

- "Three echoes gathered around home, tenderness, and missing someone."

Bad cluster sentence:

- "Your attachment wound is active again."
