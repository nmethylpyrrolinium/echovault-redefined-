# Echo Soundprint UI/UX rules

## Copy principles

- Use "echo", "orb", "soundprint", "scene", "arc", "memory", "credits", and "emotional weather" as recurring product language.
- Keep explanations short enough for cards: 80-140 characters when possible.
- Make recommendations feel chosen, not generated.
- Use "might", "leans", and "feels like" for emotional inference.
- Never shame users for repeats, sadness, anger, or nostalgia.

## Interface patterns

- Playlist card: mood title, orb color, 3-5 visible tracks, emotional arc label, one-line reason.
- Mood Receipt block: compact soundtrack, top song/category, emotional job, visual tag.
- Wrapped module: cinematic sections with shareable images.
- Feedback controls: "more like this", "less like this", "too intense", "save to my soundprint".

## Visual system

- Grief: blue-violet, rain grain, soft glow.
- Ache: rose-blue, mirror/glass, close framing.
- Joy: gold, bloom, kinetic particles.
- Awe: silver/gold, cosmic scale, wide shot.
- Anger: red-orange, high contrast, angular motion.
- Calm: aqua, spacious layout, low contrast.
- Numb: gray-blue, fog, reduced saturation.
- Confidence: emerald/gold, spotlight, sharp typography.
