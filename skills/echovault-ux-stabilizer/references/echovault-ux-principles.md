# EchoVault UX Principles

EchoVault should feel like a calm cinematic emotional universe, not a generic dashboard, CRM, notes app, or journaling clone.

## Preserve

- Emotional language: memory, vault, echo, reflection, presence, signal, ritual, return.
- Cinematic atmosphere: depth, soft contrast, spacious timing, intentional quiet.
- Trust: no surprise data loss, no dead controls, no trapped overlays, no unreadable text.
- Premium restraint: fewer visible controls, clearer hierarchy, more reliable states.

## Stabilize First

- Fix broken click/tap targets before adding new UI.
- Fix scroll and focus traps before styling overlays.
- Fix data persistence and safe fallbacks before changing onboarding.
- Fix contrast and layout overflow before adding animation.

## Premium Heuristics

- One primary action per view; secondary actions are clear but quieter.
- Copy should reassure, not explain implementation details.
- Surfaces should have consistent radius, border opacity, shadow softness, and spacing rhythm.
- Empty states should invite the next meaningful action, not shame the user for having no data.
