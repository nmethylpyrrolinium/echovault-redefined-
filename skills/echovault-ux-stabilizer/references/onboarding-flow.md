# Onboarding Flow

EchoVault onboarding should feel like entering a private emotional space, not completing admin setup.

## Recommended Sequence

1. Welcome: concise identity promise and one primary action.
2. Personalization: optional name/tone/context; skip must be allowed.
3. Privacy reassurance: explain local/cloud storage honestly in plain language.
4. First meaningful action: create/import/reflect without too many fields.
5. Empty state: invite return and continuity.

## Copy Rules

- Use gentle, cinematic verbs: begin, return, preserve, listen, gather, unlock.
- Avoid clinical productivity language unless the feature requires it.
- Keep permissions contextual: ask only when the user understands the benefit.
- Make every step skippable or reversible unless required for core function.
